# Fase 0 - Descubrimiento

Fecha de inicio: 2026-05-25

Este paquete convierte la idea inicial en una base de trabajo validable. La fase 0 no busca construir todavia, sino reducir incertidumbre antes de invertir en diseno y desarrollo.

## Objetivo

Definir con claridad:

- Que problema resuelve la aplicacion.
- Para que usuarios se construye primero.
- Que entra y que no entra en el MVP.
- Que datos federativos son necesarios y como se podrian obtener.
- Que riesgos legales, operativos y de seguridad hay que tratar desde el inicio.
- Que preguntas debemos validar con clubes, jugadores y federaciones.

## Documentos

- `01-vision-producto.md`: vision, propuesta de valor y objetivos.
- `02-usuarios-y-flujos.md`: usuarios principales y recorridos iniciales.
- `03-alcance-mvp.md`: alcance recomendado para el primer producto.
- `04-datos-federativos.md`: estrategia para datos de RFEF y federaciones territoriales.
- `05-riesgos-legales-y-seguridad.md`: riesgos de privacidad, menores, verificacion y moderacion.
- `06-validacion-y-entrevistas.md`: guion para entrevistas y criterios de validacion.
- `07-decisiones-pendientes.md`: decisiones que necesitamos cerrar antes de fase 1.

## Resultado esperado de la fase

Al terminar esta fase deberiamos tener:

- Segmento inicial elegido.
- Comunidad autonoma o zona piloto definida.
- Alcance MVP aprobado.
- Lista de datos federativos imprescindibles.
- Estrategia legal y de moderacion inicial.
- 10 a 20 entrevistas realizadas.
- Primer backlog de fase 1.

## Duracion sugerida

2 semanas.

La fase puede cerrarse antes si ya tenemos acceso a clubes piloto y suficiente claridad de datos. Tambien puede extenderse si el acceso federativo resulta complejo o si el producto necesita trabajar con menores desde el primer lanzamiento.
